# 최종 점검표

최종 검토, 교정 근거 제시, 의미 변경 위험 판단이 필요할 때 이 참고 파일을 사용한다.

## 1. 문서 제출 전 점검

### 1.0 쉬운 공문서 3요건

- [ ] 쉽게 썼는가: 어려운 말, 줄임말, 외국어, 한자어, 긴 문장을 줄였는가?
- [ ] 국민의 처지에서 표현했는가: 고압적 표현, 차별적 표현, 불필요한 인적 정보를 줄였는가?
- [ ] 정확하게 썼는가: 맞춤법, 띄어쓰기, 표준어, 외래어 표기, 오탈자, 문법, 의미 관계가 맞는가?

### 1.1 사실과 정보

- [ ] 문서 목적이 분명한가?
- [ ] 독자가 해야 할 행동이 분명한가?
- [ ] 독자의 배경지식과 예상 질문을 고려했는가?
- [ ] 날짜, 시간, 금액, 수량, 단위가 정확한가?
- [ ] 기관명, 인명, 직위, 제품명, 법령명이 정확한가?
- [ ] 출처가 필요한 정보에 출처를 달았는가?
- [ ] 불확실한 내용에 `확인 필요` 표시를 했는가?

### 1.2 문장

- [ ] 주어와 서술어가 호응하는가?
- [ ] 한 문장에 너무 많은 정보가 들어 있지 않은가?
- [ ] 수식 관계가 모호하지 않은가?
- [ ] 조사와 어미를 지나치게 생략하지 않았는가?
- [ ] 피동문과 번역 투를 줄였는가?
- [ ] 대등한 표현끼리 연결했는가?

### 1.3 단어와 표현

- [ ] 어려운 한자어를 쉬운 말로 바꾸었는가?
- [ ] 불필요한 외국어를 쉬운 우리말로 바꾸었는가?
- [ ] 일본어 투 표현을 피했는가?
- [ ] 고압적·권위적 표현을 피했는가?
- [ ] 차별적 표현이나 배제적 표현이 없는가?
- [ ] 같은 개념을 같은 용어로 썼는가?

### 1.4 구조와 형식

- [ ] 제목만 보고 문서 내용을 짐작할 수 있는가?
- [ ] 단락이 논리적 순서로 배열되어 있는가?
- [ ] 목록과 표를 적절히 사용했는가?
- [ ] 전달 매체에 맞게 정보량과 형식을 조정했는가?
- [ ] 서식은 입력 순서, 선택 항목, 작성 방법이 명확한가?
- [ ] 날짜, 시간, 항목 번호, 붙임, 끝 표시가 형식에 맞는가?
- [ ] 문의처, 담당자, 다음 단계가 필요한 경우 포함되어 있는가?

### 1.5 공고문과 홍보 글

- [ ] 공고문은 대상, 자격, 기간, 제출 방법, 선정 기준, 결과 발표, 문의처가 빠짐없는가?
- [ ] 공고문은 지원 절차 순서대로 정보를 배열했는가?
- [ ] 누리소통망서비스 홍보 글은 핵심 메시지와 행동 지침이 바로 보이는가?
- [ ] 이미지나 카드 뉴스 문안은 자세한 설명으로 연결되는 주소나 문의처를 포함했는가?

## 2. 수정 이유 분류

사용자가 교정 근거를 요청하면 다음 분류를 사용한다.

- `맞춤법`: 맞춤법, 표준어, 표기 오류
- `띄어쓰기`: 띄어쓰기 오류
- `낱말`: 단어 선택, 용어 통일, 쉬운 말, 외국어, 일본어 투 표현
- `표현`: 문장 호응, 번역 투, 피동 표현, 모호한 수식 관계, 어조
- `구성`: 문단 배열, 제목, 목록, 표, 정보 순서, 독자의 행동 흐름
- `사실 확인`: 출처, 날짜, 수치, 법령명, 기관명, 인명, 직위, 제품명, 불확실한 정보

## 3. 의미 변경 위험 알림

다음에 해당하는 수정은 사용자에게 의미 변경 가능성을 알린다.

- 법률적 효력이나 의무가 달라질 수 있는 문구
- 책임, 사과, 귀책, 인정 여부가 달라질 수 있는 문구
- 기한, 자격, 범위, 금액, 단위, 지급 조건이 달라질 수 있는 문구
- 계약, 정책, 고용, 조달, 공고와 관련된 문구
- 기술명, 제품명, API명, 법령명, 기관명, 공식 직위처럼 임의 변경하면 안 되는 명칭

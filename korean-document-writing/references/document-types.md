# 문서 유형별 작성 지침

공문, 협조 요청문, 보도 자료, 보고서, 안내문, 공고문, 업무 이메일, 회의록, 매뉴얼처럼 문서 유형별 형식이 필요할 때 이 참고 파일을 사용한다.

## 1. 공문·기안문·협조 요청문

목적: 기관 또는 조직의 의사 결정, 요청, 통보, 협조를 명확히 전달한다.

필수 요소:

- 수신자
- 제목
- 근거 또는 배경
- 목적
- 요청 사항
- 기한
- 첨부물
- 문의처

작성 지침:

- 제목은 문서의 핵심 행위를 드러내게 쓴다. 예: `○○ 협조 요청`, `○○ 제출 안내`, `○○ 결과 보고`
- 첫 문단에 근거와 목적을 밝힌다.
- 요청 사항은 번호나 표로 구체화한다.
- 상대가 해야 할 행동과 기한을 분명히 쓴다.
- `만전을 기하여 주시기 바랍니다`, `착오 없으시기 바랍니다` 같은 상투적·권위적 표현은 피한다.
- `붙임` 표시는 본문 끝 다음 줄에 쓰고, 첨부물이 여러 개면 번호를 붙인다.

권장 구조:

```markdown
수신  ○○○
제목  ○○ 협조 요청

1. 관련 근거 또는 배경입니다.
2. ○○을 위해 다음과 같이 협조를 요청드립니다.

  가. 요청 사항:
  나. 제출 기한:
  다. 제출 방법:
  라. 문의처:

붙임  1. ○○ 서식 1부
      2. ○○ 안내문 1부.  끝.
```

## 2. 보도 자료

목적: 언론이 쉽게 보도할 수 있도록 정책, 사업, 행사, 성과를 객관적으로 정리한다.

필수 요소:

- 제목: 핵심 뉴스 가치가 드러나야 한다.
- 부제: 수치, 일정, 대상, 특징을 보완한다.
- 첫 문단: 누가, 무엇을, 언제, 어디서, 왜, 어떻게 했는지 제시한다.
- 본문: 배경, 주요 내용, 의미, 기대 효과를 순서대로 설명한다.
- 인용문: 필요한 경우 책임자 발언을 간결하게 넣는다.
- 자료 출처: 통계, 연구, 외부 자료는 출처를 밝힌다.
- 문의처: 담당 부서와 연락처를 넣는다.

작성 지침:

- 객관성, 신뢰성, 공정성을 유지한다.
- 홍보성 과장 표현을 줄이고 검증 가능한 사실을 앞세운다.
- 문장은 짧게 쓰고, 첫 문단만 읽어도 핵심을 알 수 있게 한다.
- 외국어와 전문 용어는 쉬운 말로 바꾸거나 설명을 붙인다.

권장 구조:

```markdown
# 제목
## 부제

□ ○○기관은 2026. 7. 4. ○○을 시행한다고 밝혔다.

□ 이번 조치의 주요 내용은 다음과 같다.
  - 첫째, ...
  - 둘째, ...
  - 셋째, ...

□ 이번 조치로 ...할 것으로 기대된다.

담당: ○○부서 ○○팀 / 연락처: 000-0000-0000
```

## 3. 보고서·검토서

목적: 의사 결정자가 상황을 빠르게 이해하고 판단할 수 있도록 사실, 분석, 쟁점, 대안을 구조화한다.

필수 요소:

- 제목
- 요약
- 배경
- 현황 또는 문제
- 분석 또는 검토 결과
- 대안
- 권고안 또는 다음 단계
- 부록 또는 근거 자료

작성 지침:

- 첫 부분에 결론이나 핵심 요약을 먼저 쓴다.
- 사실과 의견을 구분한다.
- 판단 근거를 함께 제시한다.
- 수치, 표, 비교 기준을 명확히 쓴다.
- 추상적 표현보다 구체적·개별적 표현을 사용한다.
- `검토 필요`, `확인 필요`, `결정 필요` 항목을 명확히 표시한다.

권장 구조:

```markdown
# ○○ 검토 보고서

## 1. 요약
- 결론:
- 핵심 근거:
- 결정 필요 사항:

## 2. 배경

## 3. 현황

## 4. 쟁점

## 5. 대안 비교
| 대안 | 장점 | 단점 | 비용/일정 | 위험 |
|---|---|---|---|---|

## 6. 권고안

## 7. 다음 단계
```

## 4. 안내문·공지문

목적: 독자가 필요한 행동을 쉽게 이해하고 수행하도록 돕는다.

필수 요소:

- 안내 대상
- 핵심 안내 내용
- 일시와 장소
- 절차
- 준비물 또는 유의 사항
- 문의처

작성 지침:

- 독자가 해야 할 일을 먼저 쓴다.
- 단계별 절차를 번호로 제시한다.
- 위험, 제한, 예외는 눈에 띄게 구분한다.
- 고압적인 명령형 대신 정중하고 분명한 표현을 쓴다.
- 안전 안내문은 행동 지침을 구체적으로 쓴다.

권장 구조:

```markdown
# ○○ 안내

○○을 다음과 같이 안내드립니다.

## 대상

## 일정

## 이용 방법 / 신청 방법
1.
2.
3.

## 유의 사항
-

## 문의
```

## 5. 공고문

목적: 특정 사안, 모집, 선정, 입찰, 채용, 변경 사항 등을 대중에게 공식적으로 알린다.

필수 요소:

- 공고명
- 공고 목적
- 대상 또는 자격
- 기간
- 신청 또는 제출 방법
- 선정 기준 또는 절차
- 유의 사항
- 문의처

작성 지침:

- 시행 주체와 시행 내용을 분명히 쓴다.
- 제목만으로 무엇에 대한 공고인지 알 수 있게 쓴다.
- 복잡한 정보는 대상, 자격, 활동 내용, 기간, 접수 방법, 결과 발표, 문의처처럼 항목을 나누어 제시한다.
- 지원 절차의 흐름에 따라 정보를 배열한다.
- 추가 문의를 할 수 있도록 담당 부서와 연락처를 넣는다.
- 자격, 기한, 제출 서류, 평가 기준은 표로 정리한다.
- 법적 효력이 있는 문구는 임의로 완화하거나 바꾸지 않는다.
- 모호한 표현인 `등`, `기타`, `필요시`는 가능한 한 구체화한다.

## 6. 민원 문서용 서식

목적: 민원인이 행정기관에 허가, 인가, 신고, 증명, 교부, 처분 등 특정 행위를 요청할 수 있게 한다.

필수 요소:

- 서식명
- 신청 대상 또는 신청 자격
- 신청인이 적어야 할 정보
- 선택 항목과 공개 여부
- 첨부 서류
- 제출 방법
- 처리 기한 또는 수수료
- 작성 방법과 문의처

작성 지침:

- 민원인이 직접 쓰는 문서이므로 질문, 항목명, 작성 방법을 쉬운 말로 쓴다.
- 어려운 법률 용어, 한자어, 축약어는 풀어 쓰거나 괄호 안에 설명한다.
- 입력 칸의 순서를 실제 작성 흐름과 맞춘다.
- 선택지는 중복되거나 빠진 항목이 없게 정리한다.
- 법적 근거와 처벌 조항은 정확히 유지하되, 설명 문장은 이해하기 쉽게 바꾼다.
- 서식 안에서 같은 대상을 부르는 말은 하나로 통일한다.

## 7. 업무 이메일

목적: 수신자가 빠르게 이해하고 답하거나 조치할 수 있게 한다.

권장 구조:

```markdown
제목: [요청/공유/확인] 핵심 내용

안녕하세요, ○○님.

○○ 관련하여 아래와 같이 요청드립니다.

1. 요청 사항:
2. 기한:
3. 참고 자료:

확인 후 회신 부탁드립니다.
감사합니다.
```

작성 지침:

- 제목에 목적을 표시한다. 예: `[확인 요청]`, `[공유]`, `[검토 요청]`
- 본문 첫 문단에 용건을 쓴다.
- 요청 사항과 기한을 분리한다.
- 감정적 표현, 과장, 책임 전가성 문구를 피한다.

## 8. 회의록

목적: 회의에서 결정된 사항과 후속 조치를 정확히 기록한다.

필수 요소:

- 회의명
- 일시, 장소, 참석자
- 안건
- 논의 내용
- 결정 사항
- 할 일, 담당자, 기한

권장 구조:

```markdown
# ○○ 회의록

- 일시:
- 장소:
- 참석자:
- 안건:

## 1. 논의 내용

## 2. 결정 사항

## 3. 후속 조치
| 할 일 | 담당자 | 기한 | 상태 |
|---|---|---|---|
```

## 9. 매뉴얼·사용자 안내서

목적: 독자가 작업을 정확히 따라 할 수 있게 한다.

작성 지침:

- 작업 전 조건, 권한, 준비물을 먼저 쓴다.
- 절차는 순서대로 번호를 붙인다.
- 오류 상황과 해결 방법을 함께 적는다.
- 버튼명, 메뉴명, 파일명, 명령어는 일관된 형식으로 표시한다.
- 기술 용어는 처음 나올 때 설명한다.

권장 구조:

```markdown
# ○○ 사용 방법

## 1. 개요
## 2. 사전 준비
## 3. 절차
1.
2.
3.
## 4. 오류 해결
## 5. 문의
```

## 10. 누리소통망서비스 홍보 글

목적: 누리소통망서비스, 카드 뉴스, 짧은 이미지 문안으로 정책, 서비스, 행사, 안전 정보를 빠르게 알린다.

필수 요소:

- 핵심 메시지
- 대상 또는 조건
- 주요 행동 지침
- 기간, 장소, 금액 등 필수 수치
- 자세한 내용을 볼 수 있는 연결 주소 또는 문의처

작성 지침:

- 이미지 안에는 핵심 내용만 넣고, 보충 설명은 연결 주소나 본문에 둔다.
- 한 문장은 짧게 쓰고, 표기 오류와 맞춤법 오류를 특히 꼼꼼히 확인한다.
- 수치, 비율, 거리, 기간처럼 행동에 영향을 주는 정보는 단위까지 분명히 쓴다.
- 홍보 문구라도 과장하거나 사실을 단정하지 않는다.
- 매체별 이용자층과 접근성을 고려해 쉬운 말, 충분한 대비, 읽기 쉬운 줄바꿈을 사용한다.

권장 구조:

```markdown
# 핵심 메시지

- 대상:
- 기간:
- 해야 할 일:
- 유의 사항:
- 자세히 보기 / 문의:
```
